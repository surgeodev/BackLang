🚀 QUICK START (5 MINUTES)
═════════════════════════════════════════════════════════════════════════════

1. NAVIGATE TO EXTENSION
   cd vscode-extension

2. INSTALL DEPENDENCIES
   npm install

3. COMPILE TYPESCRIPT
   npm run compile

4. TEST IN VS CODE
   • Open the vscode-extension folder in VS Code
   • Press F5 to launch extension test window
   • Open a .bl file to see syntax highlighting
   • Try debugging features!

5. EXPLORE FEATURES

   • Ctrl+Shift+R  →  Run the file

💻 DEVELOPMENT COMMANDS
═════════════════════════════════════════════════════════════════════════════

# Install dependencies
npm install

# Compile TypeScript to JavaScript
npm run compile

# Watch for changes (auto-recompile)
npm run watch

# Check code quality
npm run lint

# Run tests
npm run test

# Create .vsix package
npm run package

# Or use vsce directly:
npm install -g vsce
vsce package


📦 PUBLISHING TO MARKETPLACE
═════════════════════════════════════════════════════════════════════════════

1. Create Package:
   npm install -g vsce
   vsce package
   → Creates backlang-debug-1.0.0.vsix

2. Publish to Marketplace:
   vsce publish
   → Makes extension available on VS Code marketplace

3. Or use the provided script:
   chmod +x publish.sh
   ./publish.sh
